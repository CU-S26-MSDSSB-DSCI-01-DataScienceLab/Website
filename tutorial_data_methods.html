Data

Data “cleaning”

  • I do not like the term “cleaning”. It is more about understanding the data and making it usable for your analysis. Lets call it data wrangling or data preparation.
  • It is often the most time-consuming part of a data science project!
  • For most project: Work on a data preparation script in a coordinated way. The scipt
    1. imports the raw data
    2. produces data frames you need for analysis
    3. writes these out to import them in your analysis and reporting files.
  • Important skill to learn: Which operation should go into the data preparation script and which should be done in the analysis script?
  • The scripts leading from raw data to the final report are sometimes called the data pipeline.
  • What to do with missing data? It depends! Find the appropriate way to deal with them for each of your specific purposes. Want to impute data? My advice is: Do not.

Machine Learning

Machine Learning is very broad

  • Machine Learning is a very broad field.
  • It is always about an algorithm that learns from data in a certain way. Often this is about fixing weights.
  • There are machine learning algorithms for classification, regression, clustering, dimensionality reduction, and so on.
  • Machine learning is also the backbone of generative AI tools.
  • For your project you will probably work on machine learning for more classical prediction problems for classification or regression.

Prediction models

  • You can always compare the performance of your machine learning model to a simple baseline model.
    • For the regresssion problem (prediciting a numerical value), a simple baseline is the linear model
    • For the classification problem (predicting a category), a simple baseline is the logistic regression.
  • If your model does not perform better than the baseline, you have not done anything wrong. You have just not found a better model.
  • If your model performs better than the baseline, you have done something right. You have found a better model. It may be better in predicting the data, but it may not be better in understanding the data.

Doing machine learning

  • A lot of computer science machine learning is about finding better learning algorithms. However for data science projects it is often more important to find better features.
  • For data science most work goes into
    • feature engineering and data preparation. The machine learning part is often just a small part of the project.
    • Validation. You need to make sure that your model generalizes to new data and does not just memorize the training data. Cross-validation is a common technique for this.
  • For reporting of machine learning results, it is important to
    • Explain the prediction problem first!
    • Understand the particular challenges for your type of problem. (For example: class imbalance for classification problems.)
    • Have a baseline model for comparison.
    • Report the performance of your model on a test set that was not used for training.
    • Understand the performance measures you use and report them correctly.

Tutorial for Independent Study

USA baby names from Google Cloud BigQuery (SQL)

This tutorial is for the independent study project. It shows how to retrieve data from the internet using SQL queries on Google Cloud BigQuery. The data is about baby names in the USA.

SQL is a language to query databases. You should learn some basics sometimes, because every larger amount of data is stored in such databases.

Preliminaries: install.packages("bigrquery") (It is assumed you have the tidyverse package.)

USA Names dataset in BigQuery

Input: Databases and SQL

  • Structured Query Language (SQL) is a language to query relational databases.
  • A relational database
    1. presents data to users in the form of a collection of tables with rows and columns, and
    2. provides relational operators to manipulate the data in tabular form.
  • SQL is somehow a standard but may come in different flavors
  • Some call it “S,Q,L”, other “SEQUEL”

Input: Database = data frame?

  • Tables in a database are indeed essentially the same as data frames in R, or python-pandas.

What is the different use cases?

  • SQL databases are
    • for storing and querying large amounts of data
    • often being access through a network
    • often so large to be distributed over several severs
  • The data frames in our data science tools are for smaller datasets to manipulate and analyze then within a programming environment and typically in memory.

Let’s do a SQL query on USA Names!

On the BigQuery interface The tab for the table usa_1910_2013 shows

  • The table Schema which shows names and data types of the columns.
    What are the columns in USA Names?
  • Some Details about the data in the table: Metadate and data about Storage
  • A preview of the first rows of the table
  1. Select “Query” –> “In split tab” to open a new tab for our query.
  2. We want to extract the numbers of babies with the name “Kevin” for each year in the whole USA.
  3. Copy this query
SELECT year, sum(number) AS num
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE name = "Kevin"
GROUP BY year
  1. Click “Run” and explore the results.

Explanations

SELECT year, sum(number) AS num
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE name = "Kevin"
GROUP BY year
  • Line 1 specifies what columns we want to have. It also specifies that we compute a new variable num which is the sum of number. To that end, we need some GROUP BY which comes later in Line 4.
  • Line 2 specifies the data table. Note the dot-notation with the database collection bigquery-public-data, the database usa_names, and the table usa_1910_2013
  • Line 3 specifies a filter
  • The line breaks are not relevant for the execution.
  • The UPPERCASE of the syntax words is not necessary and only a convention.

Query from R

Learing SQL

Suppose we have the data frame usa_names in R or python. Then the same data wrangling operations are quite similar.

SQL

SELECT year, sum(number) AS num
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE name = "Kevin"
GROUP BY year

R dplyr

usa_names |> 
filter(name == "Kevin") |> 
select(year, number) |> 
group_by(year) |> 
summarize(num = sum(number))

py pandas

usa_names.loc[usa_names['name'] == 'Kevin', 
['year', 'number']].\
groupby('year').\
agg(num=('number', 'sum'))
  • There are libraries for linking python or R to SQL databases
    • Use these when your data is in a database anyway
  • It may even make sense to make some computations in a SQL query (often much faster!)
  • SQL skills are useful (often must-have) for data scientists! Whenever there is an opportunity to learn a bit of SQL, take it!
  • Basics are not too complicated. AI tools can help to translate!
